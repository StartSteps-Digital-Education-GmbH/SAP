## Part 1: Test Script Basics

In projects, you can think of testing structure like:
- Test scenario: end-to-end business flow (high-level)
- Test case: specific check inside the scenario
- Test script: step-by-step execution with expected results

Why scripts matter:
- Consistency across testers
- Repeatable execution (especially for regression)
- Evidence generation (what happened, what the system returned)

---

## Part 2: Test Case Repository (Why It Is Needed)

A repository helps the team:
- Track test coverage
- Reuse scripts/cases in later cycles
- Demonstrate traceability from requirements → tests → results

---

## Part 3: Where Automation Fits (High-Level)

Automation can later help:
- Re-running tests faster
- Reducing manual effort
- Improving reliability of repeated regression checks

For now, focus on understanding the “why,” not tool details.
