# Group exercise: Joule patterns in a fictional rollout

**2 groups**, **2–3 members** each. Roles: **Facilitator**, **Recorder**, **Presenter**.

No SAP access required—this is a **paper workshop**.

---

## Scenario (same for both groups)

**Contoso Foods** is rolling out SAP cloud solutions. Leadership wants “AI everywhere” and asks you to propose **Joule** use cases for:

- **Procurement clerks** (high volume, many occasional users)  
- **Finance controllers** (strict controls, month-end pressure)

---

## Group 1 — Procurement clerks

Deliver **4** use cases:

- For each use case, label the **primary interaction pattern** (Transactional / Navigational / Informational / Analytical).  
- For each use case, list **one benefit** and **one risk/guardrail**.

| # | Use case (short) | Primary pattern | Benefit | Risk / guardrail |
|---|------------------|-----------------|---------|------------------|
| 1 | | | | |
| 2 | | | | |
| 3 | | | | |
| 4 | | | | |

**Presenter:** Which use case would you pilot first, and why?

---

## Group 2 — Finance controllers

Deliver **4** use cases (same table format).

**Extra requirement:** At least **two** use cases must explicitly mention **human oversight** (why it is needed).

**Presenter:** What would you **not** automate with conversational AI in month-end close, and why?

---

## Debrief prompts (class discussion)

- Where did groups disagree on **pattern labels**? Why is that OK?  
- What is one **policy** you would recommend before enabling transactional patterns?
