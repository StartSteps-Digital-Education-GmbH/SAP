# Day 4: Joule from a consultant’s perspective
## Discovery, value, and safe adoption (no systems required)

This file helps you facilitate **customer conversations** and **project thinking** about Joule **without** logging into SAP.

---

## Learning objectives

By the end of this session, you will be able to:

- Run a short **discovery** agenda for Joule (questions, not slides)
- Separate **hype** from **pilot-ready** use cases
- Connect Joule to **change management**, **security**, and **testing** concepts you already learned in implementation weeks

### 4) Adoption plan (15 min)

Minimum viable adoption includes:

- **Communications** (what changes for users, what does not)
- **Training** (how to verify answers, when to escalate)
- **Support** (who handles “Joule said X” incidents)
- **Feedback loop** (how improvements get prioritized)

---

## Consultant pitfalls (avoid these)

- Promising features **not released** for the customer’s product/edition
- Skipping **security** because “it’s just UI”
- Treating AI answers as **audit evidence** without process definition
- Ignoring **language/locale** needs in global rollouts
