# (Optional) Exercise: Create a Simple Test Case Template

This is an **optional** exercise. The goal is to design a template that makes test cases consistent and reusable.

---

## Part 1: Draft Your Template

Create a simple test case template with the fields you think are essential.

Suggested fields (you can change/add):
- Test Case ID
- Scenario / Process area
- Title / Description
- Preconditions (data, roles, environment)
- Steps
- Expected results
- Actual results
- Evidence (link/screenshot/log)
- Defect link (if any)
- Status (Pass/Fail/Blocked)
