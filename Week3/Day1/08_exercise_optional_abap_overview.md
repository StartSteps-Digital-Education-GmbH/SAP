## What Is ABAP?

**ABAP** (Advanced Business Application Programming) is SAP’s primary programming language. It is used to write **custom logic**, **reports**, **enhancements**, and **integrations** that run inside or alongside SAP applications (e.g. SAP S/4HANA, SAP ECC).

- ABAP runs on the **SAP application server** and can read and write SAP data, call business logic, and extend standard SAP behavior.
- Many SAP modules (Finance, Sales, HR, etc.) are built with ABAP; when a company needs something that standard SAP does not provide “out of the box,” developers often use ABAP to build it.

---

## Where Does ABAP Fit in SAP Application Development?

| Context | Role of ABAP |
|---------|------------------|
| **Custom reports** | Writing programs that read SAP data and produce reports (e.g. for finance, logistics). |
| **Enhancements** | Adding logic to standard SAP processes without changing SAP’s core code (e.g. user exits, BAdIs). |
| **Integrations** | Building interfaces (e.g. IDoc processing, BAPI calls) so SAP can exchange data with other systems. |
| **New applications** | Developing new transactions or apps that use SAP data and run on the SAP platform. |

ABAP is one of the **components** of SAP Application and Development Models (alongside Fiori, HANA Development, and Cloud development). Fiori often provides the **user interface**, while ABAP can provide the **backend logic** or extensions.
