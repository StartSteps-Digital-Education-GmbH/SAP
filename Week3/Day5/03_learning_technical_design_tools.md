## Part 1: What Is a “Technical Design”?

A **technical design** describes **how** the SAP solution will be built and configured, including:
- System and **technical architecture**
- Configuration settings and parameters
- **Integrations** and interfaces
- Custom developments and enhancements

It takes the **functional design** and turns it into:
- Concrete **system changes**
- Clear instructions for developers and basis/technical teams

## Part 2: Key Technical Design Tools (High‑Level)

Here are some tools and platforms that often appear in technical design work.

| Tool | Short description | Typical use in projects |
|------|-------------------|-------------------------|
| **SAP Business Application Studio / SAP Web IDE** | Cloud / web‑based development environments for SAP | Develop Fiori apps, extensions, and services |
| **SAP Solution Manager / SAP Cloud ALM** | Tools for managing and monitoring SAP systems across the lifecycle | Documentation, change management, monitoring, testing support |
| **SAP Integration Suite** | Integration platform for SAP and non‑SAP systems | Design and run interfaces, APIs, and integration flows |
| **SAP HANA tools (Studio, Web IDE for HANA, HANA cockpit)** | Tools for modeling, administration, and performance analysis in HANA | Design data models, views, and monitor performance |

In Week 4 you will see more detail on how these tools fit into **technical design documentation** and **implementation**.

## Part 3: How Technical Design Connects to Blueprint and Fiori

From Week 3, Day 3 you learned about:
- **Solution blueprint** (business processes, scope, integrations)
- **Functional and technical specifications**

Technical design:
- Uses the blueprint as a **starting point**
- Adds more detailed **technical decisions**, such as:
  - Which integration technology will be used (e.g. Integration Suite)
  - Which apps or extensions will be built using Fiori and BTP
  - How performance and security will be handled

For example:
- A blueprint might say: “Managers approve purchase orders via Fiori.”
- The technical design will describe:
  - Which **Fiori app** or custom app is used
  - How the app accesses **backend services**
  - Any **authorizations** needed
